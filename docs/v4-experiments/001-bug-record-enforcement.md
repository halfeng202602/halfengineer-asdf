# 001: バグ記録の強制化（タスク1）

> 実施日: 2026-02-18

## やったこと

### implementer.md の修正
- 行動原則に2項目追加:
  - 「実装中にエラーが発生した場合、修正前に bugs/{task-id}-{連番}.md を作成すること。記録なしでタスクをDONEにしてはならない」
  - 「バグがゼロの場合は bugs/{task-id}-no-bugs.md に「バグなし」と明示記録する」
- バグ記録セクションを「必須」に強化:
  - バグなしの場合のテンプレートを追加
  - 「bugs/ に該当タスクの記録ファイルが1つも存在しない状態でタスクをDONEにすることは禁止」を明記

### reviewer.md の修正
- レビュー観点に「7. バグ記録」を追加: bugs/ に該当タスクのバグ記録が存在するかチェック
- 完了条件に追加: 「bugs/ に該当タスクのバグ記録が存在するか確認済み。なければ REQUEST_CHANGES」

### orchestrate.md の修正
- Phase 3 の完了条件に追加: 「bugs/{task-id}-*.md が1つ以上存在すること」

### CLAUDE.md の修正
- フェーズ遷移ルールにバグ記録存在確認を追加（実装→レビュー遷移時）

### Hooks（pre-commit）
- タスク0で作成済みの pre-commit-bug-check.sh が二重強制の役割を担う

## 起きたこと

- 既存の bugs/_templates/bug-report-template.md がすでに存在していた（v3 から引き継ぎ）
- v3 の implementer.md にもバグ記録セクションはあったが「記録する」という推奨に留まっていた → v4 で「記録しないとDONEにできない」に強化
- バグゼロの場合の明示記録（no-bugs.md）は v4 の新規追加。「記録行為自体を必須化」することで空になる問題を根本解決

## 強制化の三重防御

| レイヤー | 仕組み | チェックタイミング |
|---|---|---|
| 1. implementer | 行動原則で禁止 | タスク完了時 |
| 2. reviewer | レビュー観点でチェック | レビュー時 |
| 3. pre-commit Hook | bugs/ のファイル存在チェック | コミット時 |

- 特に pre-commit Hook の実動作テスト（bugs/ 空の状態でコミットしようとしてブロックされるか）
